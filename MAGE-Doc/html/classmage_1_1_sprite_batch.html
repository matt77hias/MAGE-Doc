§ FlushBatch()

void mage::SpriteBatch::FlushBatch ( )
private

Flushes a batch of sprites for rendering if non-immediate rendering is required for the current batch of sprites. Otherwise, the sprites in the current batch are rendered immediately.

Sprites are sorted based on the sprite sorting mode and adjacent sprites are grouped for rendering if sharing the same texture.

Note
This functionality is only used if non-immediate rendering is required.

§ GrowSortedSprites()

void mage::SpriteBatch::GrowSortedSprites ( )
private

Grows the vector of sorted sprites of this sprite batch.

This effectively replaces the current vector of sorted sprites with a new vector of sorted sprites having a capacity equal to the number of sprites waiting to be drawn in the sprite queue of this sprite batch.

Note
This functionality is only used if non-immediate rendering is required.

§ GrowSpriteQueue()

void mage::SpriteBatch::GrowSpriteQueue ( )
private

Grows the sprite queue of this sprite batch.

This effectively replaces the current sprite queue with a new sprite queue having a capacity equal to two times the capacity of the current sprite queue of this sprite batch.

§ m_sprite_srvs

std::vector< ComPtr< ID3D11ShaderResourceView > > mage::SpriteBatch::m_sprite_srvs
private

A vector containing the smart pointers of the texture shader resource views.

If each mage::SpriteInfo instance held a refcount on its texture, this could end up with many redundant AddRef and Release calls on the same object, so instead this separate collection is used to hold just a single refcount each time the associated texture is changed.
